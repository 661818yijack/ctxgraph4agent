## 2. Rust/SQLite Gotchas

### MemoryType serializes to LOWERCASE
- `MemoryType::Experience` → `"experience"` in DB (NOT `"Experience"`)
- SQL `WHERE memory_type = 'Experience'` returns ZERO rows
- Always use lowercase in WHERE clauses
- Discovered via failing test: inserted Experience episode, query found nothing

### Dynamic IN clauses need params_from_iter
```rust
// WRONG: params![] needs compile-time arity
// params![id1, id2, id3]  // can't do with Vec

// CORRECT:
let placeholders: Vec<_> = ids.iter().map(|_| "?").collect();
let in_clause = format!("({})", placeholders.join(", "));
let stmt = db.prepare(&format!("SELECT * FROM episodes WHERE id IN {}", in_clause))?;
let rows = stmt.query_map(
    rusqlite::params_from_iter(ids.iter().map(|id| id as &dyn rusqlite::ToSql)),
    |row| ...
)?;
```
Pattern already exists in sqlite.rs (`get_entities_by_ids`) — copy from there.

## 3. Data Model Reference

```
episodes(id, content, source, recorded_at, memory_type, ...)
episode_entities(episode_id, entity_id)   ← junction table
entities(id, name, entity_type, memory_type, ttl_seconds, ...)
edges(id, head_id, tail_id, relation, episode_id, ...)
```

- Episode → Entity: many-to-many via `episode_entities` junction
- Edge → Episode: direct `episode_id` FK (no junction)
- get_pattern_candidates correct flow:
  1. Load Experience episodes (WHERE memory_type = 'experience')
  2. Load entity_ids via episode_entities per episode
  3. Load full Entity objects by ID
  4. Load edges WHERE episode_id IN (...)
  5. Group entities by episode_id into Vec<Vec<Entity>>
  6. Pass (episodes, entities_grouped, edges) to PatternExtractor::extract()
